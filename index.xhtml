LSPU Clinic System

  • Project Context: Group project requirement at LSPU to build a clinic information and management system for campus use.
  • Role: Frontend layout + backend integration, including CRUD pages and record/schedule database flow.
  • Main Problem: Manual logs were slow, records were difficult to track, and schedule updates were not centralized.
  • What We Built: Registration, record updates, clinic schedule management, and a readable UI for clinic staff.
  • Outcome: Faster retrieval of records, less repeated entry, and stronger full-stack teamwork experience.

Delivery Management Platform

  • Project Context: Client/customer-style requirement to solve real delivery workflow issues through a web system.
  • Role: Backend + frontend integration for role-based navigation, tracking pages, and status update flow.
  • Main Problem: Fragmented process, weak visibility for owners, unclear tracking for customers, and inefficient driver updates.
  • What We Built: One platform for customers, owners, and drivers with role-based access and clearer status progression.
  • Outcome: Better communication between roles, less tracking confusion, and a more structured operations process.

Asteroid-Empire (Wave Defense Game)

  • Project Context: Personal game-style web experiment focused on interactive UI and arcade pacing.
  • Role: Designed the game interface, implemented wave progression behavior, and tuned feedback loops.
  • Main Problem: Keep gameplay simple enough for first-time players while still feeling intense and responsive.
  • What I Built: Weapon selection flow, wave trigger actions, visual emphasis for key controls, and replay-ready pacing.
  • Outcome: A playable browser mini-game that showcases frontend logic, UI polish, and game interaction structure.
